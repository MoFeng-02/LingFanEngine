# LingFanEngine.SDK 项目结构

> **设计原则**：SDK 是独立的创作工具，**不直接引用引擎核心**。SDK 负责项目创建、DSL 编辑、资源加密、构建发布。引擎通过 NuGet 包由用户项目引用。
>
> **UI 原则**：优先使用 **C# 代码构建界面**（Code-only UI），不使用 AXAML。C# 方式对 DI 更友好，编译时类型安全，且与 ViewModel/Service 的依赖注入天然契合。
>
> **AOT 原则**：AOT 是必须的。所有代码必须 AOT 兼容——不依赖反射、JSON 使用 source generator、DI 使用 MFToolkit.AutoGenerator 源生成器。

## 各层职责总览

| 层级 | 目录 | 职责 | 关键约束 |
|---|---|---|---|
| **UI 层** | `Views/` | 纯 C# 构建 UI，数据绑定 | 不含业务逻辑，用 CompiledBinding |
| **ViewModel 层** | `ViewModels/` | 协调 UI 和服务 | 不直接访问文件系统/进程 |
| **服务抽象层** | `Services/Abstractions/` | 接口定义 | 面向接口编程 |
| **服务实现层** | `Services/Implementations/` | 具体实现 | 依赖 IPlatformService 处理平台差异 |
| **平台服务层** | `SDK.Desktop/Services/` | 平台特定功能 | 运行时判断 Win/Linux/Mac |
| **DSL 分析层** | `Dsl/` | 语法验证/AST/变量/引用/高亮 | 纯内存操作，无副作用，**不产出运行时代码** |

| **工具层** | `Utils/` | 通用辅助 | 纯静态类 |

## MFToolkit 集成

### MFToolkit.AutoGenerator — 自动 DI 注入

服务类使用特性标记，源生成器自动生成 `AddAutoInjectServices()`：

```csharp

[Singleton]   // 自动注册为 Singleton
public class ProjectService : IProjectService { ... }

[Singleton]
public class DslAnalyzer : IDslAnalyzer { ... }

// 源生成器自动生成：

//         => services.AddSingleton<IProjectService, ProjectService>()
//                    .AddSingleton<IDslAnalyzer, DslAnalyzer>();

```

### MFToolkit.Routing — 页面导航

SDK 是多页面应用，用 MFToolkit.Routing 管理页面切换：

```csharp
// Navigation/RouteDefinitions.cs
public static class RouteDefinitions
{
    public static void RegisterRoutes(IRouteRegistry routes)
    {
        routes.AddRoute<ProjectPage>("/project", r => r.SetTop().SetKeepAlive());
        routes.AddRoute<StoryEditorPage>("/editor");
        routes.AddRoute<AssetManagerPage>("/assets");
        routes.AddRoute<BuildPage>("/build");
        routes.AddRoute<SettingsPage>("/settings");
    }
}

// ViewModels 中导航
public class MainWindowViewModel(IRouter router) : ViewModelBase
{
    [RelayCommand]
    private async Task NavigateAsync(string path) => await router.NavigateAsync(path);
}
```

### CommunityToolkit.Mvvm — MVVM 模式

使用 `[ObservableProperty]` 和 `[RelayCommand]` 源生成器，**不手动实现** INotifyPropertyChanged / ICommand：

```csharp
public partial class ProjectViewModel(IProjectService projectService) : ViewModelBase
{
    [ObservableProperty]
    private ObservableCollection<ProjectConfig> _projects = new();

    [RelayCommand]
    private async Task CreateProjectAsync()
    {
        var project = await projectService.CreateNewAsync("MyGame");
        Projects.Add(project);
    }
}
```

## DI 注册流程

```csharp
// ════════════════════════════════════════════════════════════════════
// Extensions/ServiceCollectionExtensions.cs
// ════════════════════════════════════════════════════════════════════
public static class ServiceCollectionExtensions
{
    public static IServiceCollection AddLingFanEngineSDK(this IServiceCollection services)
    {
        // MFToolkit.AutoGenerator 自动注入所有标记了 [Singleton]/[Scoped]/[Transient] 的服务
        services.AddAutoInjectServices();

        // 路由（MFToolkit.Routing）
        services.AddRouting();
        services.AddRoutes(routes => RouteDefinitions.RegisterRoutes(routes));

        // ViewModels（Transient，每次导航创建新实例）
        services.AddTransient<MainWindowViewModel>();
        services.AddTransient<ProjectViewModel>();
        services.AddTransient<StoryEditorViewModel>();
        services.AddTransient<AssetManagerViewModel>();
        services.AddTransient<BuildViewModel>();
        services.AddTransient<SettingsViewModel>();

        // IPlatformService 不在这里注册——由平台项目注册
        return services;
    }
}

// ════════════════════════════════════════════════════════════════════
// SDK.Desktop/Program.cs
// ════════════════════════════════════════════════════════════════════
public static class Program
{
    [STAThread]
    public static void Main(string[] args)
    {
        var services = new ServiceCollection();

        // 注册 SDK 核心服务
        services.AddLingFanEngineSDK();

        // 注册平台服务
        if (OperatingSystem.IsWindows())
            services.AddSingleton<IPlatformService, WindowsPlatformService>();
        else if (OperatingSystem.IsLinux())
            services.AddSingleton<IPlatformService, LinuxPlatformService>();
        else if (OperatingSystem.IsMacOS())
            services.AddSingleton<IPlatformService, MacOSPlatformService>();

        var provider = services.BuildServiceProvider();

        // 启动 Avalonia（纯 C# App，无 .axaml）
        var lifetime = new ClassicDesktopStyleApplicationLifetime
        {
            Args = args,
            ShutdownMode = ShutdownMode.OnLastWindowClose,
        };

        AppBuilder.Configure<App>()
            .UsePlatformDetect()
            .AfterSetup(_ => { /* 注入 DI 到 App */ })
            .SetupWithLifetime(lifetime);

        lifetime.MainWindow = new MainWindow(provider);
        lifetime.Start(args);
    }
}
```

## DSL 分析层设计（编辑时，非运行时编译）

SDK 的 DSL 分析层**不产出运行时代码**。引擎在运行时自行编译 `.story` 文件。SDK 的"编译"实际上是**加密**。

### 职责边界

| 功能 | SDK 负责 | 引擎负责 |
|---|---|---|
| DSL 语法验证 | ✅ 编辑时实时检查 | ❌ |
| AST 构建 | ✅ 用于场景树/高亮 | ❌ |
| 变量提取/展示 | ✅ 编辑器侧边栏 | ❌ |
| 跨文件引用分析 | ✅ 跳转/补全 | ❌ |
| 语法高亮 | ✅ 编辑器着色 | ❌ |
| DSL → ICommand 编译 | ❌ | ✅ 运行时 |
| .story 加密 | ✅ 构建时 | ❌ |
| .story 解密 | ❌ | ✅ 运行时 |

### 解析技术：Pidgin 解析器组合子

SDK 的 DSL 分析层**必须使用 Pidgin**（引擎已有 fork，位于 `DLL/Pidgin.dll`）进行词法和语法分析。

- **禁止使用正则表达式**（`Regex`）进行 DSL 解析——正则在复杂嵌套语法面前脆弱且不可维护
- **禁止手写状态机或递归下降解析器**——与引擎核心 `DslStatementParser` / `DslExpressionEvaluator` 保持技术栈统一
- Lexer 和 Parser 均用 Pidgin 的 `Parser<char, T>` / `Parser<Token, T>` 组合子构建
- 参考引擎核心 `DslStatementParser.cs` 的组合子写法风格

### 分析流程

```
.story 源码
    │
    ▼
Lexer（Pidgin Parser<char, Token>）→ Token 流
    │
    ▼
Parser（Pidgin Parser<Token, SyntaxNode>）→ AST（SyntaxTree）
    │
    ├──→ DiagnosticCollector → 错误/警告列表（行号+消息）
    ├──→ VariableCollector → 变量表（名称/类型/定义位置/引用位置）
    ├──→ SceneReferenceResolver → 跨文件引用表（navigate/jump/call 目标）
    └──→ Highlighter → 高亮 token 列表（关键字/字符串/注释/变量/标签）
```

## 加密流程（无清单）

```
用户执行构建
    │
    ▼
1. KeyManager 检查/生成密钥（32字节 AES-256）
   → 存储到 ~/.lingfan/keys/{项目哈希}.key
    │
    ▼
2. ResourceEncryptor 加密所有资源文件
   → .story → .story.enc（AES-256-GCM: nonce(12)+tag(16)+ciphertext）
   → 图片/音频/视频 → .enc
   → 直接输出到项目目录，不生成 manifest.json
    │
    ▼
3. KeyInjector 生成 GeneratedKeys.cs
   → 密钥分片写入多个 static readonly byte[]（防完整提取）
   → [AggressiveInlining] GetKey() 方法组合分片
   → 生成在用户项目命名空间下（如 MyGame.Security.GeneratedKeys）
    │
    ▼
4. PublishService 调用 dotnet publish（AOT）
   → 用户项目 csproj 已含 <PackageReference Include="LingFanEngine" />
   → 引擎运行时扫描 .enc 文件 → 检测加密 → 解密 → 编译执行
   → 热重载天然支持：增删改 .story.enc 文件，引擎重新扫描即可
```

### 为什么不需要清单

引擎的 `StoryRegistry.Scan()` 本身就自动扫描 `Stories/` 目录。加密后：

- **加故事**：丢个新 `.story.enc` → 扫描到 → 解密 → 编译
- **删故事**：删掉文件 → 下次扫描不在 → 自然消失
- **改故事**：文件变更 → 文件监听触发 → 重新解密编译

清单反而要维护"加了什么、删了什么"的状态一致性，纯属多余。

## 纯 C# UI 构建原则

SDK 所有界面用 C# 代码构建，不使用 `.axaml` 文件。

### 为什么选择 C# UI

1. **DI 友好**：View 构造函数直接注入 ViewModel/Service，无需 XAML 的 `DataContext` 间接赋值
2. **类型安全**：`CompiledBinding.Create` 在编译时检查绑定路径，无运行时错误
3. **重构友好**：重命名/提取方法/查找引用等 IDE 功能完整支持
4. **无 XAML 编译开销**：省去 AXAML → IL 的中间步骤

### 示例：页面构建

```csharp
// Views/Pages/ProjectPage.cs
public class ProjectPage : UserControl
{
    public ProjectPage(ProjectViewModel viewModel)
    {
        DataContext = viewModel;
        InitializeComponent();
    }

    private void InitializeComponent()
    {
        var grid = new Grid
        {
            RowDefinitions = RowDefinitions.Parse("Auto,*,Auto"),
            Margin = new Thickness(16),
        };

        // 标题
        grid.Children.Add(new TextBlock
        {
            Text = "项目管理",
            FontSize = 24,
            FontWeight = FontWeight.Bold,
        });

        // 项目列表
        var listBox = new ListBox
        {
            [!ItemsControl.ItemsSourceProperty] =
                CompiledBinding.Create<ProjectViewModel, ObservableCollection<ProjectConfig>>(
                    vm => vm.Projects),
        };
        Grid.SetRow(listBox, 1);
        grid.Children.Add(listBox);

        // 创建按钮
        var createBtn = new Button { Content = "新建项目" };
        createBtn.Bind(Button.CommandProperty,
            CompiledBinding.Create<ProjectViewModel>(
                source: (ProjectViewModel)DataContext!,
                expression: vm => vm.CreateCommand));
        Grid.SetRow(createBtn, 2);
        grid.Children.Add(createBtn);

        Content = grid;
    }
}
```

## 额外要求

**AOT 开发是必须的。**

- 不使用反射（`Type.GetType()`/`PropertyInfo.SetValue()` 等）
- JSON 序列化使用 `[JsonSerializable]` source generator
- DI 使用 MFToolkit.AutoGenerator 源生成器（编译时生成注册代码）
- 绑定使用 `CompiledBinding`（编译时验证，非 `ReflectionBinding`）
