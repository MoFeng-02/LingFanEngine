## 阶段 0：基础设施搭建（P0 — 阻塞后续所有工作）

### 0.1 AppHost + AppHostBuilder 启动框架

**要做什么**：实现 Builder 模式的 DI 容器构建框架，替代当前空桩。

**怎么开发**：

```
Bootstrapper/AppHost.cs:
  - 持有 IServiceProvider
  - 暴露 GetService<T>() / GetRequiredService<T>()
  - 实现 IDisposable（Dispose 时释放容器）
  - 静态 Current 属性供全局访问

Bootstrapper/AppHostBuilder.cs:
  - 内部 ServiceCollection
  - 链式方法：ConfigureServices(Action<IServiceCollection>)
  - 链式方法：UseLingFanEngineSDK()（调用 AddLingFanEngineSDK）
  - Build() → 返回 AppHost
```

**关键要点**：
- 直接利用 `Microsoft.Extensions.Hosting`（已引用），但不需要完整 Generic Host——只需要 `ServiceCollection` + `BuildServiceProvider()`
- AppHostBuilder 允许平台项目在 Build 前追加自己的服务注册（如 IPlatformService）

### 0.2 纯 C# App 类（替换 .axaml）

**要做什么**：将 `App.axaml` + `App.axaml.cs` 改为纯 C# 的 `App.cs`。

**怎么开发**：

```csharp
// App.cs
public class App : Application
{
    private IServiceProvider _services = null!;

    public void Initialize(IServiceProvider services)
    {
        _services = services;
        Styles.Add(new FluentTheme());
        // 注册全局 C# 样式
        RegisterGlobalStyles();
    }

    public override void OnFrameworkInitializationCompleted()
    {
        if (ApplicationLifetime is IClassicDesktopStyleApplicationLifetime desktop)
        {
            desktop.MainWindow = new MainWindow(_services);
        }
        base.OnFrameworkInitializationCompleted();
    }

    private void RegisterGlobalStyles()
    {
        // C# 代码定义全局样式（按钮/文本框等统一样式）
        Styles.Add(new Style(x => x.OfType<Button>())
        {
            Setters = { new Setter(Button.PaddingProperty, new Thickness(16, 8)) }
        });
    }
}
```

**关键要点**：
- 删除 `App.axaml` 和 `App.axaml.cs`
- FluentTheme 通过 `Styles.Add(new FluentTheme())` 添加
- 全局样式在 C# 中用 `Style` 对象构建

### 0.3 AddLingFanEngineSDK() DI 注册

**要做什么**：实现 `Extensions/ServiceCollectionExtensions.cs` 的 `AddLingFanEngineSDK()` 方法。

**怎么开发**：

```csharp
public static class ServiceCollectionExtensions
{
    public static IServiceCollection AddLingFanEngineSDK(this IServiceCollection services)
    {
        // MFToolkit.AutoGenerator 自动注入所有 [Singleton] 标记的服务
        services.AddAutoInjectServices();

        // 路由
        services.AddRouting();
        services.AddRoutes(routes => RouteDefinitions.RegisterRoutes(routes));

        // ViewModels
        services.AddTransient<MainWindowViewModel>();
        services.AddTransient<ProjectViewModel>();
        services.AddTransient<StoryEditorViewModel>();
        services.AddTransient<AssetManagerViewModel>();
        services.AddTransient<BuildViewModel>();
        services.AddTransient<SettingsViewModel>();

        return services;
    }
}
```

**关键要点**：
- 服务实现类标记 `[Singleton]`，`AddAutoInjectServices()` 自动注册
- IPlatformService 不在这里注册（由平台项目提供）

### 0.5 Desktop Program.cs 入口

**要做什么**：重写 `LingFanEngine.SDK.Desktop/Program.cs`，集成 DI + 纯 C# App。

**怎么开发**：

```csharp
public static class Program
{
    [STAThread]
    public static void Main(string[] args)
    {
        var services = new ServiceCollection();
        services.AddLingFanEngineSDK();

        // 平台服务
        if (OperatingSystem.IsWindows())
            services.AddSingleton<IPlatformService, WindowsPlatformService>();
        else if (OperatingSystem.IsLinux())
            services.AddSingleton<IPlatformService, LinuxPlatformService>();
        else if (OperatingSystem.IsMacOS())
            services.AddSingleton<IPlatformService, MacOSPlatformService>();

        var provider = services.BuildServiceProvider();

        var lifetime = new ClassicDesktopStyleApplicationLifetime
        {
            Args = args,
            ShutdownMode = ShutdownMode.OnLastWindowClose,
        };

        AppBuilder.Configure<App>()
            .UsePlatformDetect()
            .WithInterFont()
            .AfterSetup(_ =>
            {
                var app = Application.Current as App;
                app?.Initialize(provider);
            })
            .SetupWithLifetime(lifetime);

        lifetime.Start(args);
    }
}
```

## 阶段 4：DSL 分析层（P1）

### 4.1 Lexer（词法分析器）

**要做什么**：将 `.story` 源码字符串转换为 Token 流。

**实现要点**：
- 参考 DSL 语法关键字（scene/label/say/set/if/while/menu/navigate/jump 等）
- TokenType 枚举：Keyword/Identifier/String/Number/Symbol/Comment/Newline
- Token 记录：类型 + 值 + 行号 + 列号
- **必须使用 Pidgin 解析器组合子库**（引擎已有 fork，位于 `DLL/Pidgin.dll`），不使用正则表达式或手写状态机
- 与引擎核心保持一致的解析技术栈——引擎的 `DslStatementParser` 和 `DslExpressionEvaluator` 都基于 Pidgin，SDK 也用 Pidgin 保证语法一致性
- **关键**：这是 SDK 独立的分析器，与引擎的 DslStatementParser 目的不同（编辑验证 vs 运行时编译），但共用 Pidgin 解析技术

### 4.2 Parser（语法分析器）

**要做什么**：Token 流 → AST。

**实现要点**：
- SyntaxNode 类型：SceneNode/LabelNode/SayNode/SetNode/IfNode/WhileNode/MenuNode/NavigateNode/JumpNode 等
- **使用 Pidgin 解析器组合子**构建语法分析器（`Parser<Token, SyntaxNode>` 组合），与 Lexer 一致，不使用手写递归下降或其他解析技术
- 参考引擎核心 `DslStatementParser.cs` 的 Pidgin 组合子写法（如 `Try(String("say"))` / `Between(String("\""), String("\""))` 等），保持语法解析风格统一
- 语法错误 → 产出 Diagnostic（行号 + 消息 + 原始行内容）
- AST 保留完整位置信息（用于跨文件跳转）

### 4.3 VariableCollector

**要做什么**：遍历 AST 提取所有变量。

**实现要点**：
- 收集 `set "key" {expr}` / `define "key" value` / `let "key" value` 的定义
- 收集 `{variable}` / `{a + b}` 等表达式中的变量引用
- 输出 VariableInfo（名称/定义位置/引用位置列表）
- 用于编辑器侧边栏变量列表 + 变量高亮

### 4.4 SceneReferenceResolver

**要做什么**：跨文件场景/标签引用解析。

**实现要点**：
- 遍历 AST 找 `navigate "scene_name"` / `jump label` / `call label` / `scene "name"`
- 与目录中其他 .story 文件的场景定义交叉引用
- 输出 SceneReference（源文件/行号/目标场景/目标标签/引用类型）
- 用于编辑器"跳转到定义"功能

### 4.5 Highlighter

**要做什么**：AST + Token 流 → 高亮 token 列表。

**实现要点**：
- HighlightToken：起始位置 + 长度 + 高亮分类（Keyword/String/Comment/Variable/Label/Number/Symbol）
- 分类映射到颜色（在 ViewModel/View 中决定具体颜色）
- 编辑器根据 HighlightToken 列表着色渲染

## 阶段 5：加密模块（P1）

### 5.1 AesEncryptor

**要做什么**：AES-256-GCM 加密/解密。

**实现要点**：
```csharp
public static class AesEncryptor
{
    // 加密：返回 nonce(12) + tag(16) + ciphertext
    public static byte[] Encrypt(byte[] plaintext, byte[] key);

    // 解密：从 nonce + tag + ciphertext 还原明文
    public static byte[] Decrypt(byte[] encrypted, byte[] key);
}
```
- 使用 `System.Security.Cryptography.AesGcm`（AOT 友好）
- 加密时生成随机 nonce（`RandomNumberGenerator.Fill`）
- 魔数头 `0x4C46454E`（"LFEN"）+ 版本号 1 字节，用于运行时检测

### 5.2 KeyManager

**要做什么**：密钥生成 + 分片 + 本地存储。

**实现要点**：
- `GenerateKey()` → 32 字节随机密钥（`RandomNumberGenerator.Fill`）
- `SaveKey(projectPath, key)` → 存储到 `~/.lingfan/keys/{SHA256(projectPath)}.key`
- `LoadKey(projectPath)` → 读取本地密钥
- `GetOrCreateKey(projectPath)` → 有则加载，无则生成+保存
- 密钥分片：将 32 字节拆为 N 段（默认 4 段 × 8 字节），分散存储在代码不同位置

### 5.3 KeyInjector

**要做什么**：生成 `GeneratedKeys.cs` 源代码文件。

**实现要点**：
```csharp
public static string GenerateKeyFile(string namespaceName, byte[] key, int shardCount = 4)
{
    // 生成类似：
    // namespace MyGame.Security;
    // internal static class GeneratedKeys
    // {
    //     private static readonly byte[] _s1 = { 0x4A, 0x6F, ... };  // 8 bytes
    //     private static readonly byte[] _s2 = { 0x75, 0x72, ... };  // 8 bytes
    //     private static readonly byte[] _s3 = { 0x6E, 0x65, ... };  // 8 bytes
    //     private static readonly byte[] _s4 = { 0x79, 0x21, ... };  // 8 bytes
    //     [MethodImpl(MethodImplOptions.AggressiveInlining)]
    //     internal static byte[] GetKey()
    //     {
    //         var key = new byte[32];
    //         _s1.CopyTo(key, 0);
    //         _s2.CopyTo(key, 8);
    //         _s3.CopyTo(key, 16);
    //         _s4.CopyTo(key, 24);
    //         return key;
    //     }
    // }
}
```
- 文件生成在用户项目命名空间下（不是引擎命名空间）
- 用户项目启动时通过 `LingFanEngineOptions.EncryptionKey = GeneratedKeys.GetKey()` 注入
- `[AggressiveInlining]` 在 AOT 编译时内联，密钥字节折叠为机器码常量

**要做什么**：DSL 文本编辑 + 语法高亮 + 场景树 + 错误面板。

**C# UI 要点**：
- 左侧：文件树（TreeView 显示 Stories/ 目录结构）
- 中间：文本编辑区（需要自定义文本渲染控件支持高亮，或用 Avalonia 的 TextBox + 文本着色）
- 右侧上方：场景树（TreeView 从 AST 构建）
- 右侧下方：错误/警告面板（ListBox 绑定 DslAnalysisResult.Errors）
- 底部状态栏：变量数量/场景数量/文件路径

**关键挑战**：
- Avalonia 内置 TextBox 不支持语法高亮着色
- 方案 A：自定义 `TemplatedControl` 继承文本渲染，支持 Run 着色
- 方案 B：使用 `SelectingTextBlock` + `Inlines` 实现只读高亮预览
- 方案 C：集成第三方编辑器组件（如果有 Avalonia 兼容的）
- 初期可先用方案 B 做只读预览 + 普通 TextBox 做编辑，后续升级

## 阶段 8：平台服务实现（P2）

### 8.1 WindowsPlatformService

**实现要点**：
- `GetAppDataDirectory()` → `%APPDATA%\LingFan`
- `OpenInFileExplorer(path)` → `Process.Start("explorer.exe", path)`
- `GetSystemFonts()` → 扫描 `C:\Windows\Fonts` 或用 `System.Drawing.Text.InstalledFontCollection`
- 权限检查：文件读写权限

### 8.2 LinuxPlatformService

**实现要点**：
- `GetAppDataDirectory()` → `~/.local/share/LingFan`（XDG）
- `OpenInFileExplorer(path)` → `xdg-open`
- `GetSystemFonts()` → 扫描 `~/.local/share/fonts` + `/usr/share/fonts`

### 8.3 MacOSPlatformService

**实现要点**：
- `GetAppDataDirectory()` → `~/Library/Application Support/LingFan`
- `OpenInFileExplorer(path)` → `open`
- `GetSystemFonts()` → 扫描 `/System/Library/Fonts` + `~/Library/Fonts`

### 建议开发顺序

```
阶段 0 (基础设施) ──┬──→ 阶段 6 (UI 主框架) ──→ 阶段 7 (各页面 UI)
                    │
阶段 1 (模型) ──────┼──→ 阶段 3 (服务层) ─────┘
                    │         │
阶段 2 (工具) ──────┘         ├──→ 阶段 5 (加密)
                              │
阶段 4 (DSL 分析) ────────────┘
                              │
阶段 8 (平台服务) ←────────── 阶段 3.7
```

**第一步**：阶段 0 + 1 + 2 并行，搭建骨架。
**第二步**：阶段 3 + 4 + 5 并行，实现核心能力。
**第三步**：阶段 6 + 7 + 8，搭建 UI 和平台适配。
